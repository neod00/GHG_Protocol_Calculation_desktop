# Unsigned Beta Release Ops (Windows MSI)

## 1. 목적과 범위

- 대상: 파일럿 고객/초기 베타 사용자
- 배포 형태: `unsigned MSI` 직접 다운로드
- 전제: SmartScreen 경고 가능
- 목표: 비용 최소화 + 라이선스 통제 + 강제 업데이트 운영 검증

## 2. 배포 채널

- 1차 채널: Vercel 정적 다운로드 페이지
- 파일 저장: GitHub Releases 또는 Cloudflare R2
- 권장 파일명:
  - `GHG-Desktop-0.1.0-beta.1-x64.msi`
  - `GHG-Desktop-latest.json` (업데이트 메타데이터)

## 3. 다운로드 페이지 문구 정책

- 설치 전 고지(필수):
  - 베타 버전이며 코드 서명 미적용 상태
  - Windows SmartScreen 경고가 표시될 수 있음
  - 설치 중 경고 시 `추가 정보 -> 실행`으로 진행
  - 민감 데이터는 기본적으로 로컬 저장됨
- 신뢰 신호:
  - SHA-256 해시 제공
  - 릴리즈 노트 제공
  - 빌드 날짜/버전/지원 OS 명시

## 4. 설치 안내 정책 (지원 문서/FAQ)

1. 다운로드 링크 클릭
2. MSI 다운로드 완료
3. 실행 시 SmartScreen 경고가 뜨면 `추가 정보` 클릭
4. `실행` 클릭
5. 설치 완료 후 앱 실행
6. 첫 실행 시 라이선스 키 입력

## 5. 업데이트 정책

- 체크 타이밍:
  - 앱 시작 시 1회
  - 24시간마다 1회 (백그라운드)
- 모드:
  - 선택 업데이트: 새 버전 안내 + 나중에
  - 강제 업데이트: `minimumSupportedVersion` 미만은 사용 차단
- 실패 처리:
  - 업데이트 서버 미접속 시 기존 버전 유지
  - 강제 업데이트 대상은 재시도 버튼 + 종료만 허용

## 6. 라이선스 정책 (베타)

- 베타 라이선스 상태:
  - `active`, `expired`, `blocked`
- 앱 동작:
  - `active`: 정상 사용
  - `expired`: 읽기 전용 또는 제한 모드
  - `blocked`: 실행 차단 + 지원 문의 안내
- 서버로 전송하는 데이터 최소화:
  - 라이선스 키, 앱 버전, 디바이스 ID(해시), 체크 시각
  - 계산 데이터/보고서 원문은 전송 금지

## 7. 운영 체크리스트 (릴리즈 당일)

1. MSI 업로드
2. SHA-256 계산/게시
3. 릴리즈 노트 게시
4. `latest.json` 갱신
5. 라이선스 서버의 `latestVersion`/`minimumSupportedVersion` 확인
6. 새 설치 테스트
7. 기존 버전에서 업데이트 테스트
8. 차단 라이선스 동작 테스트

## 8. 롤백 정책

- 심각 장애 시:
  - `latest.json`을 직전 안정 버전으로 즉시 롤백
  - `minimumSupportedVersion`은 당일에는 올리지 않음
  - 설치 페이지 상단에 장애 공지 배너 노출

## 9. 베타 종료 조건 (코드서명 전환 시점)

- 다음 조건 중 1개 이상 충족 시 코드서명 전환:
  - 유료 고객 3개사 이상
  - 월 활성 사용자 30명 이상
  - SmartScreen 경고로 인한 설치 실패 문의가 월 5건 이상
